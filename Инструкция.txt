Настройка сборки:
1. Репозиторий git@github.com:saby-integration/v8unpack.git выкачиваем, устанавливаем по инструкции для cmd, или через pip install. Так же добавлен как сабмодуль.
	pip install v8unpack
	pip install v8unpack --upgrade для обновления версии
2. Устанавливаем сабмодуль core в ту же ветку, что и основной репозиторий, выкачиваем изменения:
3. Собираем файл обработки - build.cmd, в результате получаем в папке bin epf файл обработки.
4. Делаем исправления, дебажим и прочее в конфигураторе 1С.
5. Разбираем файл обработки - extract.cmd, в результате исправления разъедутся по файлам репозитория и/или сабмодуля.
6. Commit в основной репозитоприй и core, если затрагивалось.


Структура каталога:

\rep
	\bin	
	\core
	\local
	\src
	index.json 
	build.cmd	
	extract.cmd	
	update_index.cmd	
	
bin - папка куда собирается/разбирается обработка
core - папка сабмодуля с общими файлами для 8.2 и 8.3
src - папка с уникальными файлами обработки, вспомогательные файлы необходимые для сборки
local - папка с выносом областей, где содержаться особенности платформы.


Если при доработке добавлялись новые формы, то 
	1 extract.cmd
	2 update_index.cmd
	3 Результат: в конец index.json добавились новый файлы и пути для сборки/разборки
		При необходимости корректируем index.json: если файл должен браться из src, а не core то путь должен быть пустым для файлов .json ({ИмяНовойФормы}.json и {ИмяНовойФормы}.form803.json)
	4 повторно extract.cmd, если правился index.json
		Результат: разбирается обработка с новыми настройками из index.json, заменяются файлы в src и core.
	5 commit
